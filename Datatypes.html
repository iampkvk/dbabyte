List of Data Types and Their Usage in Oracle Database

In Oracle Database, various data types are available to store different kinds of data. Here's a comprehensive list of the data types along with their usage:

1. Numeric Data Types

  • NUMBER(p,s): Stores fixed and floating-point numbers.
    • p: Precision (total number of digits).
    • s: Scale (number of digits to the right of the decimal).
  • FLOAT: A subtype of NUMBER, representing floating-point numbers.
  • INTEGER: A subtype of NUMBER that represents whole numbers (equivalent to NUMBER(38,0)).
  • SMALLINT: A subtype of NUMBER for smaller integer values (equivalent to NUMBER(5)).
  • DECIMAL(p,s): Similar to NUMBER, used for fixed-point numbers.

2. Character Data Types

  • CHAR(n): Fixed-length character strings (up to 2000 bytes).
  • VARCHAR2(n): Variable-length character strings (up to 4000 bytes). Recommended for most applications due to efficiency.
  • NCHAR(n): Fixed-length national character set strings (up to 2000 bytes).
  • NVARCHAR2(n): Variable-length national character set strings (up to 4000 bytes).
  • CLOB: Character Large Object for storing large amounts of text (up to 4 GB).

3. Binary Data Types

  • BLOB: Binary Large Object for storing large binary data (up to 4 GB).
  • RAW(n): Stores raw binary data (up to 2000 bytes).
  • LONG RAW: For larger raw binary data (up to 2 GB), but less commonly used due to limitations.

4. Date and Time Data Types

  • DATE: Stores date and time values (includes year, month, day, hour, minute, second).
  • TIMESTAMP: Stores date and time with fractional seconds (up to 9 digits).
  • TIMESTAMP WITH TIME ZONE: Includes time zone information.
  • TIMESTAMP WITH LOCAL TIME ZONE: Stores time in the sessionâ€™s time zone, converted to UTC for storage.
  • INTERVAL YEAR TO MONTH: Represents a period of time in years and months.
  • INTERVAL DAY TO SECOND: Represents a period of time in days, hours, minutes, and seconds.

5. LOB Data Types

  • CLOB: For large text data (character data).
  • BLOB: For large binary data (images, videos).
  • NCLOB: National Character Large Object for large amounts of text in a national character set.

6. Object Data Types

  • OBJECT: Allows the creation of user-defined data types that can contain multiple attributes (similar to classes in OOP).
  • TABLE: Allows for creating nested tables as a type.

7. JSON Data Type

  • JSON: Oracle Database supports storing JSON data, often using VARCHAR2 or CLOB, but it also has specific features for querying JSON data.

8. Spatial Data Types

  • SDO_GEOMETRY: Used for storing spatial data (geographic objects) in Oracle Spatial.

9. XML Data Types

  • XMLTYPE: Used for storing XML data and provides methods for querying and manipulating XML.

10. Miscellaneous Data Types

  • ROWID: A unique identifier for each row in a table.
  • UROWID: A variable-length row identifier for tables or indexes.

Summary

Choosing the right data type is crucial for optimizing performance, storage, and data integrity in Oracle databases. Each data type has its specific use cases, so it's essential to understand their characteristics and applications when designing your database schema.
